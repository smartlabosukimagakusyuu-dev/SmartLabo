# 62. CEO Publish Guide — 公開手順書（スクリーンショット不要）

このドキュメントは、Smart Labo Worksホームページを正式公開するために、**CEOご自身が
行う作業**を順番にまとめたものです。専門用語は最小限にし、迷わず進められるよう、
各ステップに「何をするか」「なぜ必要か」「確認方法」を記載しています。

**前提**：会社設立・XServer契約・`smartlaboworks.com`のドメイン設定・
`form.smartlaboworks.com`サブドメインの作成・無料SSLの有効化は、
すでに完了しています。ここから先の8ステップが残りの作業です。

---

## 全体の流れ

```
①GitHub Pages切替 → ②DNS確認 → ③XServerアップロード → ④config.php設定
→ ⑤reCAPTCHA設定(後回し可) → ⑥送信テスト → ⑦公開確認 → ⑧公開後確認
```

順番どおりに進めてください。①②が終わるまでは、ホームページ本体（`smartlaboworks.com`）
は正式ドメインで表示されません。③〜⑥が終わるまでは、問い合わせフォームは実際には
動作しません（見た目は表示されますが、送信するとエラーになります）。

---

## ①GitHub Pages切替

**何をするか**：GitHubのリポジトリ設定で、ホームページの公開先を、正式ドメイン
`smartlaboworks.com`に切り替えます。

**手順**：
1. GitHubで`SmartLabo`リポジトリを開く
2. 上部メニューの「Settings」をクリック
3. 左側メニューの「Pages」をクリック
4. 「Custom domain」という欄に `smartlaboworks.com` と入力し、「Save」をクリック
5. 少し待つと、「Enforce HTTPS」というチェックボックスが選択できるようになるので、
   チェックを入れる（自動でSSL証明書が発行されます。数分〜数十分かかる場合があります）

**確認方法**：Pages設定画面に緑色のチェックマークと「Your site is live at
https://smartlaboworks.com/」という表示が出れば完了です。

---

## ②DNS確認

**何をするか**：ドメイン`smartlaboworks.com`が、正しくGitHub Pagesを指しているか確認します（多くの場合、①より前にすでに設定済みのはずですが、念のため確認します）。

**手順**：
1. ドメインを管理している画面（Xserverのドメイン管理、または別のDNS管理サービス）を開く
2. `smartlaboworks.com`（設定によっては`@`と表記）に、以下4つのAレコードが登録されているか確認する
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`
3. `www.smartlaboworks.com`に、CNAMEレコード（`smartlabosukimagakusyuu-dev.github.io`）が登録されているか確認する
4. `form.smartlaboworks.com`が、XServerのサーバーを指すレコードになっているか確認する（すでに設定完了と伺っています）

**確認方法**：上記の設定になっていればOKです。もし設定が異なる、またはよくわからない場合は、無理に変更せずご相談ください（DNSの誤設定はサイト全体がアクセスできなくなるリスクがあります）。

---

## ③XServerアップロード

**何をするか**：問い合わせフォームのプログラム（PHP）一式を、XServerへアップロードします。

**手順**：
1. XServerのファイルマネージャー（またはFTPソフト）を開く
2. `form.smartlaboworks.com`のフォルダ（ドキュメントルート、通常`public_html`という名前）を開く
3. パソコン内の `xserver-form/public/` フォルダの中身を**すべて**、そのままアップロードする
   （`contact.php`・`csrf-token.php`・`.htaccess`・`lib`フォルダ一式）
4. `public_html`フォルダの**1つ上の階層**（`public_html`と同じ場所にある、その親フォルダ）に、
   `config.php`というファイルをアップロードする（このファイルは次の④で作成します）

**注意**：`config.php`は`public_html`の**中に入れないでください**。中に入れてしまうと、
誰でもブラウザから中身を見られる状態になり危険です。

---

## ④config.php設定

**何をするか**：③でアップロードした`config.php`に、実際のメール・セキュリティ設定を書き込みます。

**手順**：
1. パソコン内の `xserver-form/private/config.php.example` というファイルをコピーし、
   ファイル名を `config.php` に変更する
2. `config.php`をテキストエディタで開き、以下の値を埋める

   | 項目 | 埋める値 |
   |---|---|
   | `smtp_host` | XServerのメール設定画面に表示されているSMTPサーバー名（例：`sv〇〇.xserver.jp`） |
   | `smtp_port` | `587`のままでよい |
   | `smtp_encryption` | `'tls'`のままでよい |
   | `smtp_user` | `info@smartlaboworks.com` |
   | `smtp_pass` | `info@smartlaboworks.com`のメールパスワード |
   | `admin_email` | `info@smartlaboworks.com`（問い合わせ通知を受け取るアドレス） |
   | `csrf_secret` | ランダムな文字列（下記コマンドで生成） |
   | `recaptcha_enabled` | `false`のままでよい（⑤で変更） |
   | `recaptcha_secret` | 空のままでよい（⑤で変更） |

3. `csrf_secret`の生成方法：パソコンでコマンドプロンプト（またはターミナル）を開き、
   `php -r "echo bin2hex(random_bytes(32));"` と入力してEnterを押すと、
   ランダムな文字列が表示されます。それを`csrf_secret`に貼り付けてください。

4. 保存した`config.php`を、③の手順で`public_html`の1つ上の階層へアップロードする

**確認方法**：次の⑥「送信テスト」で正しく動作するかを確認します。

---

## ⑤reCAPTCHA設定（後回しにしても構いません）

**何をするか**：問い合わせフォームへのbot（自動スパム送信）対策を、より強力にします。
**この手順を飛ばしても、フォーム自体は動作します**（すでに他の4つの対策で守られています）。手が空いたときで構いません。

**手順**：
1. [Google reCAPTCHA管理画面](https://www.google.com/recaptcha/admin)を開き、Googleアカウントでログイン
2. 新しいサイトを登録する。「reCAPTCHA v3」を選択し、ドメインに`smartlaboworks.com`を入力
3. 発行された「サイトキー」と「シークレットキー」の2つをメモする
4. パソコン内の `WEBSITE/contact.html` を開き、コメントアウトされている
   `<script src="https://www.google.com/recaptcha/api.js?render=YOUR_SITE_KEY">` の
   `YOUR_SITE_KEY` をサイトキーに置き換え、コメント（`<!--` と `-->`）を外す
5. `WEBSITE/js/pages.js` の `RECAPTCHA_SITE_KEY = ""` の `""` の中に、同じサイトキーを入力する
6. `config.php`の`recaptcha_enabled`を`true`に、`recaptcha_secret`にシークレットキーを設定する
7. 変更したファイルをGitHubへコミット・push（または担当エンジニアに依頼）し、XServerへも
   `config.php`を再アップロードする

---

## ⑥送信テスト

**何をするか**：実際に問い合わせフォームから送信し、正しくメールが届くか確認します。

**手順**：
1. ブラウザで `https://smartlaboworks.com/contact.html` を開く
2. すべての項目に、テスト用の内容を入力する（自分宛のメールアドレスを使う）
3. 「送信する」ボタンを押す
4. 「お問い合わせを受け付けました」という完了画面が表示されることを確認する
5. `info@smartlaboworks.com`のメールボックスを確認し、通知メールが届いているか確認する
6. 3で入力したメールアドレスにも、自動返信メールが届いているか確認する

**うまくいかない場合**：完了画面が出ずエラーが表示される場合は、④の`config.php`の
設定値（特にSMTPパスワード）が正しいか再確認してください。それでも解決しない場合は、
担当エンジニアにエラーメッセージをそのまま伝えてください。

---

## ⑦公開確認

**何をするか**：ホームページ全体が、正式ドメインで正しく表示されるか確認します。

**手順**：
1. ブラウザで `https://smartlaboworks.com/` を開き、トップページが表示されることを確認する
2. アドレスバーに鍵マーク（🔒）が表示され、証明書エラーが出ないことを確認する
3. ヘッダーメニューから「製品」「機能」「不動産向け」「導入プラン」「会社情報」を
   それぞれクリックし、すべて正しく表示されることを確認する
4. スマートフォンでも同様にアクセスし、表示が崩れていないことを確認する

---

## ⑧公開後確認

**何をするか**：公開直後に発生しやすい問題がないか、最終確認します。

**手順**：
1. 存在しないページ（例：`https://smartlaboworks.com/abc123`）にアクセスし、
   「ページが見つかりません」という案内が表示されることを確認する
2. SNS（X/Facebook等）でURLをシェアしてみて、タイトル・説明文・画像が正しく
   表示されるか確認する（実際に投稿する必要はなく、下書き画面での確認で構いません）
3. 数日後、Google検索で「Smart Labo Works」と検索し、サイトが表示され始めるか確認する
   （検索エンジンへの反映には数日〜数週間かかることがあります）
