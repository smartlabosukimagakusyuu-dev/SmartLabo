# 99_CEO_MEMORY

**代表(CEO)の記憶を蓄積する場所**

---

## このフォルダの目的

`00_Foundation` が対外的にも通用する「会社としての公式見解」だとすれば、`99_CEO_MEMORY` は、そこに至るまでの**代表個人の生の記憶**を蓄積する場所です。

- 番号を `99` としているのは、`00`(会社の土台)から始まったすべての物語が、最終的にここに集約・蓄積されていく、という位置づけを表しています。
- ここに書かれた内容は、時間が経つほど価値が増します。忙しさを理由に記録を後回しにせず、重要な出来事があったときはできるだけ早く書き残してください。
- 内容が十分に固まった・言語化できたものは、[00_Foundation/](../00_Foundation/README.md) 側の該当ドキュメント(Company Story、Founder Message、Brand Identity など)に反映し、洗練された「公式見解」として昇華させてください。

---

## 00_Foundation との違い

| | 00_Foundation | 99_CEO_MEMORY |
|---|---|---|
| 性質 | 公式・洗練された見解 | 個人的・生々しい記録 |
| 想定読者 | 社員・AI・外部パートナー全員 | 主に社内、将来的な自分自身 |
| 更新頻度 | 大きな意思決定があったときのみ | 出来事があるたびに随時 |
| 文体 | 整えられた文章 | 率直なメモ・箇条書きでよい |

## 利用ルール

- **このフォルダは代表本人が書く(または代筆を承認する)ことを前提とします。** AIエージェントが代筆する場合は、必ず本人の確認を経てから確定させてください。
- 完璧な文章を書こうとしないでください。箇条書き・断片的なメモでも構いません。重要なのは記録が残ることです。
- 機密性の高い内容(取引先の内部情報、未公開の資金調達の詳細など)を含む場合は、リポジトリの公開範囲に注意してください。非公開で管理すべき内容は、このリポジトリではなく安全な手段で別途管理することも検討してください。

---

## Version管理

- 各ファイルは追記型(過去の記録を消さず、下に積み重ねていく)を基本とします。
- 大きな節目(資金調達、大型契約、組織の転換点など)ごとに、ファイル内の記録に日付を明記してください。

---

## 更新ルール

- 随時更新して構いません。「まとまってから書こう」ではなく、思いついたときに書き残すことを推奨します。
- 内容が十分に固まったら、[00_Foundation/](../00_Foundation/README.md) 側への反映を検討してください。

---

## Git運用ルール

- 通常のドキュメントと同様、直接コミットで構いません。
- 機密情報を含む可能性があるため、コミット前に公開範囲を必ず確認してください。
