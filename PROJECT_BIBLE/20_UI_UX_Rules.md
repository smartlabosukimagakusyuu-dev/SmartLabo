# 20. UI/UX Rules — デザイン思想と原則

## デザインの核となる5つのキーワード

> **シンプル・余白・高級感・信頼感・ミニマル**

Smart Labo が手がけるすべてのUI(プロダクト・サイト・資料)は、この5つを基準に設計・レビューされます。派手さではなく、長く愛されるブランドを設計することを目指します(参照: [00_Foundation/07_Brand_Identity.md](00_Foundation/07_Brand_Identity.md))。

---

## 参照する4社とそこから学ぶこと

| 企業 | 具体的に取り入れる要素 |
|---|---|
| **Apple** | 不要な装飾を削ぎ落とす。一画面の情報量を絞る。動きは控えめかつ滑らかに |
| **Notion** | 情報の階層をわかりやすく。柔軟だが迷わないナビゲーション |
| **Linear** | キーボード操作を含む高速な操作性。無駄のないコンポーネント設計 |
| **Stripe** | 信頼感のあるタイポグラフィとドキュメント構成。配色は抑制的に |

---

## 具体的なデザイン原則

### 1. 余白を恐れない
- 情報を詰め込みすぎない。余白そのものがデザインの一部であると考える。
- 1画面・1セクションで伝えるメッセージは絞る。

### 2. 配色はミニマルに
- ベースは白・黒・グレーの階調を基本とし、アクセントカラーは1〜2色に絞る。
- 具体的なカラーコードは [BRAND/](../BRAND/README.md) で管理する。

### 3. タイポグラフィで高級感を出す
- 装飾に頼らず、フォントサイズ・行間・余白のバランスで「高級感」を表現する。
- 見出しと本文のコントラストを明確にする。

### 4. 動き(モーション)は控えめに
- アニメーションは「意味のある動き」のみ採用する(状態変化の説明など)。
- 過剰な演出・自動再生は避ける。

### 5. 毎日使いたくなる体験を優先する
- 起動の速さ、画面遷移の少なさ、心地よい手触りにこだわる([00_Foundation/06_Philosophy.md](00_Foundation/06_Philosophy.md) 原則3)。
- 「たまに使う業務システム」ではなく、社長が毎日自然に開きたくなるプロダクトを目指す。

### 6. 一貫性を保つ
- ボタン・フォーム・カードなどのUIパーツは、プロダクト・サイトをまたいで統一されたスタイルを使う。
- 新しいコンポーネントを作る前に、既存のデザインシステム([DESIGN/](../DESIGN/README.md))を確認する。

---

## レビュー時のチェックリスト

新しい画面・資料をリリースする前に、以下を確認してください。

- [ ] 情報を詰め込みすぎていないか(余白は十分か)
- [ ] 配色はブランドカラーの範囲内か
- [ ] 不要な装飾・アニメーションが入っていないか
- [ ] スマートフォン・PC双方で崩れなく表示されるか
- [ ] 毎日使っても疲れない、心地よい体験になっているか
- [ ] Apple / Notion / Linear / Stripe のいずれかに「似た質の高さ」を感じられるか
