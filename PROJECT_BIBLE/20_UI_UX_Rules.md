# 20. UI/UX Rules — デザイン思想と原則

## デザインの核となる5つのキーワード

> **シンプル・余白・高級感・信頼感・ミニマル**

Smart Labo が手がけるすべてのUI(プロダクト・サイト・資料)は、この5つを基準に設計・レビューされます。派手さではなく、長く愛されるブランドを設計することを目指します(参照: [00_Foundation/07_Brand_Identity.md](00_Foundation/07_Brand_Identity.md))。

---

## 参照する5社とそこから学ぶこと

| 企業 | 具体的に取り入れる要素 |
|---|---|
| **Apple** | 不要な装飾を削ぎ落とす。一画面の情報量を絞る。動きは控えめかつ滑らかに |
| **Notion** | 情報の階層をわかりやすく。柔軟だが迷わないナビゲーション |
| **Linear** | キーボード操作を含む高速な操作性。無駄のないコンポーネント設計 |
| **Stripe** | 信頼感のあるタイポグラフィとドキュメント構成。配色は抑制的に |
| **Porsche** | 性能や機能を誇示しない佇まい。時代に流されない、静かな説得力 |

---

## 具体的なデザイン原則

### 1. 余白を最優先する
- 情報を詰め込みすぎない。余白そのものがデザインの一部であると考える。
- 1画面・1セクションで伝えるメッセージは絞る。
- 何を足すかより、何を削れるかを先に考える。

### 2. 配色はSmart Blueのみをアクセントに
- ベースはWhite / Blackを基本とし、**アクセントカラーはSmart Blueのみ**とする(複数のアクセントカラーを併用しない)。
- グラデーションは必要最小限に留め、装飾目的では使用しない。
- 具体的なカラーコードは [BRAND/](../BRAND/README.md) で管理する。

### 3. タイポグラフィで高級感を出す
- 装飾に頼らず、フォントサイズ・行間・余白のバランスで「高級感」を表現する。
- 見出しと本文のコントラストを明確にする。
- 読みやすさをデザインの美しさより先に確認する。

### 4. 動き(モーション)は静かに
- アニメーションは「意味のある動き」のみ採用する(状態変化の説明など)。
- 過剰な演出・自動再生は避ける。
- 動きは**ゆっくり・自然・静か**に。驚かせるための演出、速く派手なアニメーションは使わない。

### 5. ボタンは小さく、美しく
- CTAボタンは小さめに、美しい角丸で。主張しすぎず、それでいて迷わず見つけられる配置にする。
- ボタンの数は絞る(1画面・1セクションで主役となるCTAは基本的に1つ)。

### 6. 情報量より理解しやすさ
- たくさん伝えるより、確実に伝わることを優先する。
- 機能ではなく価値を伝える。「何ができるか」ではなく「何が変わるか」を語る。

### 7. 毎日使いたくなる体験を優先する
- 起動の速さ、画面遷移の少なさ、心地よい手触りにこだわる([00_Foundation/06_Philosophy.md](00_Foundation/06_Philosophy.md) 原則3)。
- 「たまに使う業務システム」ではなく、社長が毎日自然に開きたくなるプロダクトを目指す。

### 8. 一貫性を保つ
- ボタン・フォーム・カードなどのUIパーツは、プロダクト・サイトをまたいで統一されたスタイルを使う。
- 新しいコンポーネントを作る前に、既存のデザインシステム([DESIGN/](../DESIGN/README.md))を確認する。

### 9. 禁止するビジュアル表現を使わない
- AIロボット、サイバー演出、ネオン、未来都市CG、青い地球、六角形、基板イラスト、過度なエフェクト、派手なアニメーション、情報過多のUIは使用しない。
- 推奨ビジュアル(朝のオフィス、自然光、ガラス・木目、美しいPC、ノート、デスク、人の手、洗練された空間)を優先する。
- 詳細は [PROMPTS/DESIGN/SmartLabo_Design_Bible.md](../PROMPTS/DESIGN/SmartLabo_Design_Bible.md) を参照。

## レビュー時のチェックリスト

新しい画面・資料をリリースする前に、以下を確認してください。

- [ ] 情報を詰め込みすぎていないか(余白は十分か)
- [ ] 配色はWhite / Black / Smart Blueの範囲内か
- [ ] 不要な装飾・アニメーションが入っていないか(動きはゆっくり・自然・静かか)
- [ ] ボタンは小さめ・美しい角丸で、CTAが1つに絞られているか
- [ ] 禁止ビジュアル(ロボット・サイバー・ネオン・六角形・基板等)を使っていないか
- [ ] スマートフォン・PC双方で崩れなく表示されるか
- [ ] 毎日使っても疲れない、心地よい体験になっているか
- [ ] Apple / Notion / Linear / Stripe / Porsche のいずれかに「似た質の高さ」を感じられるか
