# 20. UI/UX Rules — デザイン思想と原則

## デザインの核となるキーワード

> **Premium・Enterprise・Professional・Minimal・Modern・Trust・Innovation・AI First・Business・Executive・Luxury**

Smart Labo が手がけるすべてのUI(プロダクト・サイト・資料)は、これらのキーワードを基準に設計・レビューされます。世界トップクラスのBtoB SaaSブランドと肩を並べる完成度を目指します(参照: [00_Foundation/07_Brand_Identity.md](00_Foundation/07_Brand_Identity.md))。

最終目標は「かっこいい」と思わせることではなく、経営者に**「この会社なら任せたい」「まずはデモを見たい」**と思わせることです。

## 具体的なデザイン原則

### 1. 余白を十分に使い、情報を整理する
- 情報を詰め込みすぎない。余白そのものがデザインの一部であると考える。
- 1画面・1セクションで伝えるメッセージは絞る。
- ただし余白を優先するあまり弱々しくならないこと。Enterprise SaaSらしい力強さ(太めの見出し、はっきりしたコントラスト、堂々としたレイアウト)と両立させる。

### 2. 配色はNavyとWhiteを基調に、Smart Blueファミリーのみをアクセントに
- 基調色は **Primary Navy `#0A1B3D`** と **White `#FFFFFF`**。
- アクセントは **Smart Blueファミリー**(Smart Blue `#2563EB` / Light Blue `#60A5FA` / Accent Blue `#00D4FF`)のみ。用途の目安は以下の通り。
  - **Smart Blue** — ボタン・リンクなど主要なCTA
  - **Light Blue** — ホバー状態、サブアクセント、チャートの補助系列
  - **Accent Blue** — グロー効果やデータ可視化のハイライトなど、ごく限定的に
- 補助グレーとして **Light Gray `#F3F6FA`**(セクション区切り)・**Gray `#64748B`**(非強調テキスト)・**Dark Gray `#1E293B`**(White背景上の本文)を使用する。
- グラデーションは「上品に」使用する(Navy→Smart Blueなど、Smart Blueファミリー内の階調のみ。派手な多色グラデーションは禁止)。
- 背景は明るさと未来感を両立させる。
- 例外として、ステータスバッジ(完了=緑・進行中=青・未着手=グレー・遅延=赤)には慣用的な意味色を使用してよい。また写真・イラストの背景に限り、朝日・夕景などの自然な暖色を認める(UIチェックロームには適用しない)。詳細は [00_Foundation/07_Brand_Identity.md](00_Foundation/07_Brand_Identity.md) の「使用カラー」を参照。
- 具体的なカラーコードは [BRAND/](../BRAND/README.md) で管理する。

### 3. タイポグラフィで高級感と信頼感を出す
- フォントサイズ・行間・余白のバランスで「高級感」「信頼感」を表現する。
- 見出しと本文のコントラストを明確にする。
- 読みやすさをデザインの美しさより先に確認する。

### 4. 動きは上品に、意味を持たせる
- ホバーアニメーションなど、インタラクションに応じた動きを積極的に使う。
- ただし過剰な演出・自動再生は避け、上品さを保つ。

### 5. ボタンは大きめに、高級感を持たせる
- CTAボタンは大きめに、美しい角丸で。ホバーアニメーションを付け、高級感を演出する。
- 営業導線を意識し、複数のCTA(例: 無料デモを見る/導入相談をする/資料ダウンロード)を適切に配置する。

### 6. 機能ではなく価値を伝える
- 「何ができるか」ではなく「何が変わるか」を語る。
- 例: メール対応 → 返信漏れを防ぐ / CRM → 顧客情報を一元管理 / 契約書 → 契約作成時間を短縮 / 経営分析 → データから意思決定を支援。

### 7. 毎日使いたくなる体験を優先する
- 起動の速さ、画面遷移の少なさ、心地よい手触りにこだわる([00_Foundation/06_Philosophy.md](00_Foundation/06_Philosophy.md) 原則3)。
- 「たまに使う業務システム」ではなく、社長が毎日自然に開きたくなるプロダクトを目指す。

### 8. 一貫性を保つ
- ボタン・フォーム・カードなどのUIパーツは、プロダクト・サイトをまたいで統一されたスタイルを使う。
- 新しいコンポーネントを作る前に、既存のデザインシステム([DESIGN/](../DESIGN/README.md))を確認する。
- 基本コンポーネントの型は以下を標準とする。
  - **プライマリボタン:** Navy地または Smart Blue地の塗りつぶし、白文字、角丸
  - **セカンダリボタン:** アウトライン(枠線)、塗りつぶしなし
  - **テキストリンク:** 下線なし、Smart Blue文字、矢印アイコンを添えてよい
  - **入力フィールド:** 通常時は薄いグレー枠、フォーカス時はSmart Blueの枠線でハイライト
  - **カードコンポーネント:** タイトル・説明文・数値(例: 金額・前月比)を縦に並べた白背景カード
  - **ステータスバッジ:** 完了(緑)・進行中(青)・未着手(グレー)・遅延(赤)の4種を標準の状態表現とする
- アイコンは、AI分析・タスク管理・顧客管理・売上管理・レポート・通知・設定・セキュリティ等の代表的な機能に対応する、線画(アウトライン)スタイルの統一アイコンセットを使用する。詳細は [DESIGN/README.md](../DESIGN/README.md) を参照。

### 9. 禁止するビジュアル表現を使わない
- 安っぽいAIロボット、六角形背景、サイバー演出のやりすぎ、ネオンカラー、過度な発光、情報を詰め込みすぎるデザイン、チープなイラストは使用しない。
- 未来都市・AIネットワーク等のモチーフ自体は禁止ではない。「安っぽく」「過度に」使うことを禁止する。
- 推奨ビジュアル(未来都市、高層ビル、朝日、自然光、都市ネットワーク、AIネットワーク〈控えめに〉、洗練されたオフィス、ガラス、金属、実際のSmart Labo Works画面)を優先する。
- 詳細は [PROMPTS/DESIGN/SmartLabo_Design_Bible.md](../PROMPTS/DESIGN/SmartLabo_Design_Bible.md) を参照。

## Design Assets運用ルール

ロゴ・背景画像・アイコン・ダッシュボードのモックアップ・ボタン/カードの素材・グラフィック・写真・カラースウォッチ・フォントなど、**デザイン資産の実データは [DESIGN_ASSETS/](../DESIGN_ASSETS/README.md) に一元管理します**(CEO指示、2026-07-03)。

- `DESIGN_ASSETS/` は Logo / Background / Icons / Dashboard / Buttons / Cards / Graphics / Images / Colors / Fonts の10サブフォルダで構成されます。
- このドキュメント(思想・原則)と `DESIGN_ASSETS/`(実データ)の役割を混同しないこと。新しいUI原則やカラールールを追加する場合は、まずこのドキュメント([00_Foundation/07_Brand_Identity.md](00_Foundation/07_Brand_Identity.md) と本ファイル)を更新し、そのうえで `DESIGN_ASSETS/` に対応する実データを追加してください。
- 実データを使用する前に、必ず本ドキュメントの禁止ビジュアル・カラールールに沿っているか確認してください。
- 詳細な運用ルール(命名規則・バージョニング・Git運用)は [DESIGN_ASSETS/README.md](../DESIGN_ASSETS/README.md) を参照してください。

## レビュー時のチェックリスト

新しい画面・資料をリリースする前に、以下を確認してください。

- [ ] 情報を詰め込みすぎていないか(余白は十分か)
- [ ] 配色はNavy / White / Smart Blueファミリー(Smart Blue・Light Blue・Accent Blue)/ グレー階調の範囲内か
- [ ] グラデーションは上品か(派手な多色グラデーションになっていないか)
- [ ] ボタンは大きめ・美しい角丸で、ホバーアニメーションが付いているか
- [ ] 禁止ビジュアル(安っぽいロボット・六角形・過度なサイバー演出・ネオン・過度な発光)を使っていないか
- [ ] 機能の羅列ではなく、価値が伝わる見せ方になっているか
- [ ] スマートフォン・PC双方で崩れなく表示されるか
- [ ] 毎日使っても疲れない、心地よい体験になっているか
- [ ] Apple級の完成度・Stripe級の美しさ・Microsoft級の信頼感・OpenAI級の先進性を感じられるか
- [ ] 見た経営者が「この会社なら任せたい」「デモを見たい」と思える仕上がりか
