# 12. Pricing Philosophy — Smart Labo Works 導入プラン思想

このドキュメントは、Smart Labo Worksの「料金」に関する正式な思考の枠組みを定義します。2026-07-12のCEO指示により、**ホームページ・Product Book・Company OS・営業資料・見積書・契約書のすべてが、この共通仕様に従う**ことが正式に決定されました。

---

## 目的

料金体系をバラバラに運用せず、どの資料・どの画面で提示しても矛盾のない「ひとつの導入プラン思想」として統一することを目的とします。

---

## 基本思想

**「料金」ではなく「導入プラン」として表現する。**

単なる価格表ではなく、「お客様の会社に合わせて、どう導入するか」を一緒に設計するという姿勢を、呼称・構成の両面で一貫させます。以降、社内外を問わず「料金プラン」という表現は避け、「導入プラン」を正式な呼称とします。

**Smart Labo Worksは、中核機能を細かく切り売りする製品ではない。** 企業規模と活用範囲に応じて、必要な業務モジュールと導入支援を組み合わせる、企業向けAI業務支援SaaSである。Company Brain・CRM・AI Assistantといった中核機能は「追加すれば使えるオプション」ではなく、選択したプランに標準搭載される「利用モジュール」として扱い、真に個別性の高い支援・拡張のみを「追加オプション」として区別する（2026-07-13 CEO指示による整理、詳細は下記変更履歴参照）。

---

## 4つの構成要素

導入プランは、以下の4要素の組み合わせで構成されます（2026-07-13、③オプションを③利用モジュール／④追加オプションへ分割）。

### ① 利用規模

顧客企業の規模に応じた区分。

| 区分 | 想定 |
|---|---|
| **Small** | 小規模企業（目安：〜10名程度） |
| **Medium** | 中規模企業（目安：11〜50名程度） |
| **Enterprise** | 大規模企業（目安：51名以上、複数拠点等） |

> 具体的な人数区分・料金額は未確定。ヒアリングを通じて個別に確認する運用とし、本書では「目安」として記載する。

### ② 基本プラン

利用できる機能範囲に応じた区分。

| 区分 | 想定 |
|---|---|
| **Lite** | 主要機能を体験できる入門プラン |
| **Standard** | Smart AI Router・業種特化テンプレートを含む標準プラン |
| **Premium** | 専用カスタマイズ・優先サポートを含む最上位プラン |

> ①利用規模とは独立した軸である。「Enterprise規模の顧客がLiteプランを選ぶ」といった組み合わせも起こりうる。

### ③ 利用モジュール

**選択した基本プランに標準搭載される中核機能。** 単体の追加オプションとしては提示せず、プランに応じて自動的に含まれる（顧客向けUIでは選択・変更不可のチェック済み表示とする）。

- Company Brain
- CRM
- AI Assistant
- 案件管理
- 契約管理
- AIテンプレート

> プランごとにどのモジュールを含めるかの正式な配分は未確定。`WEBSITE/js/configurator.js`のマスターデータ（`PLAN_MODULES`）のみを変更すれば全画面へ反映される構造とし、本書では確定額・確定配分を先取りして記載しない。

### ④ 追加オプション

基本プラン・利用モジュールに含まれない、個別性の高い追加支援・拡張機能。

- Meeting／議事録AI
- OCR
- 外部API連携
- AI活用研修
- Company Brain初期構築支援
- オリジナルテンプレート制作
- データ移行
- 導入・運用支援

> 各利用モジュール・追加オプションの提供状況（実装済み／Coming Soon／要個別相談）は、`smartlabo-works/PRODUCT_BOUNDARY.md`の実装状況分類に従う。未実装の項目を「現在利用可能」であるかのように表現してはならない。

## 適用範囲（共通仕様）

以下のすべてが、この「① 利用規模 × ② 基本プラン × ③ 利用モジュール × ④ 追加オプション」という同一の構造を用いること。

| 資料・画面 | 適用方法 |
|---|---|
| ホームページ（`WEBSITE/pricing.html`、"Smart Labo Configurator"） | ①②③④をStep形式のシミュレーターとして提示。具体的な金額は「個別相談」「お問い合わせ」表記とする。顧客向けUIでは「Company OS」という名称は使用せず、「Smart Labo Configurator」「Smart Labo Works」「利用モジュール」「追加オプション」の名称に統一する |
| Product Book | 同上の構成で製品資料に反映する（Product Book自体は本書執筆時点で未作成。作成時にこの構成を用いる） |
| Company OS | 下記「Pricing Manager」を通じて、社内向けに①②③④を選択・管理する（社内専用機能であり、この名称は顧客向け画面には表示しない） |
| 営業資料 | ①②③④の構成に沿って提案書・提案スライドを作成する |
| 見積書 | 選択された①②③④の組み合わせから見積り内容を構成する |
| 契約書 | 見積書の内容と整合する形で契約条件を記載する |

## Pricing Manager（Company OS 新機能・設計）

CEOより、Company OSへ「Pricing Manager」を追加し、営業担当が①利用規模→②プラン→③オプションを選択すると、料金表・見積書・契約書・営業資料を自動生成できる設計とするよう指示があった。本章ではその**設計（データモデル・画面遷移・生成対象）**を定義する。**実際のコード実装は本書の対象外であり、smartlabo-worksリポジトリ側で別途スコープを定めたTaskとしてCEO承認を得たうえで着手する。**契約書の自動生成は特に法的な正確性が求められるため、テンプレート文言の確定にはCEOの確認が必須となる。

### データモデル（設計案。2026-07-13、ホームページ"Smart Labo Configurator"実装時にmodules/optionsを分離）

```
UsageScale   { id: 'small' | 'medium' | 'enterprise', label, description }
Plan         { id: 'lite' | 'standard' | 'premium', label, description, modules: ModuleId[], highlights[] }
Module       { id, label, status: 'ready' | 'planned' | 'consult' }   // ③利用モジュール
Option       { id, label, category, status: 'ready' | 'planned' | 'consult' }   // ④追加オプション

ProposalSelection {
  usageScale: UsageScaleId
  plan: PlanId
  modules: ModuleId[]   // 選択したPlanから自動確定(顧客は変更不可)
  options: OptionId[]   // 顧客が任意に選択
  companyName, contactName, ...(見積書・契約書に必要な顧客情報)
}
```

> `WEBSITE/js/configurator.js`の`selection`オブジェクトと同一のフィールド名・status語彙を用いる。ホームページ実装が正の実装であり、Company OS Pricing Manager実装時はこの構造をそのまま踏襲する。

### 画面遷移（設計案。2026-07-13、5Stepへ改訂）

```
① 利用規模を選択（Small / Medium / Enterprise）
    ↓
② 基本プランを選択（Lite / Standard / Premium）
    ↓
③ プランに含まれる利用モジュールを表示（選択済み・変更不可）
    ↓
④ 追加オプションを選択（複数選択可）
    ↓
⑤ おすすめ構成を確認
    ↓（ホームページでは、選択内容を引き継いでcontact.htmlへ）
Company OS Pricing Manager（将来実装）では、ここからさらに以下を選ぶ
    ├─ 料金表（PDF/画面表示）
    ├─ 見積書
    ├─ 契約書（要CEO確認済みテンプレート）
    └─ 営業資料
```

### 生成対象4種の位置づけ

| 生成物 | 用途 | 備考 |
|---|---|---|
| 料金表 | 顧客への提示用一覧 | ①②③の選択内容を表形式で出力 |
| 見積書 | 正式な見積り | 顧客情報・選択内容・金額を含む |
| 契約書 | 正式契約 | 法的テンプレートの事前確定が必須。CEO確認なしに自動生成しない |
| 営業資料 | 提案・商談資料 | ①②③の構成に沿ったスライド／PDF |

### 実装状況

**ホームページ側（`WEBSITE/pricing.html`）は"Smart Labo Configurator"として実装済み（2026-07-13）。** ①利用規模→②基本プラン→③利用モジュール表示→④追加オプション→⑤おすすめ構成確認の5Stepシミュレーターとして動作し、選択内容を`contact.html`へ引き継ぐ。見積書・契約書・提案書の自動生成、CRMへの自動登録、バックエンドAPI接続は対象外（選択内容はURLクエリ＋`localStorage`にとどまる）。

**Company OS「Pricing Manager」自体は引き続き未実装（設計のみ）。** `smartlabo-works/app.html`には現状、`PLANS`という異なる形式の静的データ（Starter/Pro/Enterprise、月額固定金額）が存在するが、これは本書の「①利用規模×②基本プラン×③利用モジュール×④追加オプション」構造とは異なり、正式な導入プラン仕様が定まる前の暫定データである。Pricing Manager実装時にこのデータを本書の構造へ置き換える、または移行方針を別途検討する必要がある。

## 金額について

本書は**構造（枠組み）を定義するものであり、具体的な金額を定めるものではない**。①②③の組み合わせごとの金額は、`BUSINESS_STRATEGY.md`の「未着手」項目として、今後CEOとの議論により確定する。金額が未確定の間は、ホームページ・営業資料等すべてにおいて「個別相談」「お問い合わせ」の表記を用いる。
