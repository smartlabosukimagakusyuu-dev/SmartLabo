# 12. Pricing Philosophy — Smart Labo Works 導入プラン思想

このドキュメントは、Smart Labo Worksの「料金」に関する正式な思考の枠組みを定義します。2026-07-12のCEO指示により、**ホームページ・Product Book・Company OS・営業資料・見積書・契約書のすべてが、この共通仕様に従う**ことが正式に決定されました。

---

## 目的

料金体系をバラバラに運用せず、どの資料・どの画面で提示しても矛盾のない「ひとつの導入プラン思想」として統一することを目的とします。

---

## 基本思想

**「料金」ではなく「導入プラン」として表現する。**

単なる価格表ではなく、「お客様の会社に合わせて、どう導入するか」を一緒に設計するという姿勢を、呼称・構成の両面で一貫させます。以降、社内外を問わず「料金プラン」という表現は避け、「導入プラン」を正式な呼称とします。

---

## 3つの構成要素

導入プランは、以下の3要素の組み合わせで構成されます。

### ① 利用規模

顧客企業の規模に応じた区分。

| 区分 | 想定 |
|---|---|
| **Small** | 小規模企業（目安：〜10名程度） |
| **Medium** | 中規模企業（目安：11〜50名程度） |
| **Enterprise** | 大規模企業（目安：51名以上、複数拠点等） |

> 具体的な人数区分・料金額は未確定。ヒアリングを通じて個別に確認する運用とし、本書では「目安」として記載する。

### ② プラン

利用できる機能範囲に応じた区分。

| 区分 | 想定 |
|---|---|
| **Lite** | 主要機能を体験できる入門プラン |
| **Standard** | Smart AI Router・業種特化テンプレートを含む標準プラン |
| **Premium** | 専用カスタマイズ・優先サポートを含む最上位プラン |

> ①利用規模とは独立した軸である。「Enterprise規模の顧客がLiteプランを選ぶ」といった組み合わせも起こりうる。

### ③ オプション

プランに追加できる個別機能・サービス。

- Company Brain構築（初期設定の代行・支援）
- OCR
- 議事録（Meeting AI）
- API連携
- AI教育（利用研修）
- テンプレート制作（業種・企業ごとの独自テンプレートの開発）
- その他

> 各オプションの提供状況（実装済み／Coming Soon／要個別相談）は、`smartlabo-works/PRODUCT_BOUNDARY.md`の実装状況分類に従う。未実装のオプションを「現在利用可能」であるかのように表現してはならない。

---

## 適用範囲（共通仕様）

以下のすべてが、この「① 利用規模 × ② プラン × ③ オプション」という同一の構造を用いること。

| 資料・画面 | 適用方法 |
|---|---|
| ホームページ（`WEBSITE/pricing.html`） | 導入プランとして①②③を紹介。具体的な金額は「個別相談」「お問い合わせ」表記とする |
| Product Book | 同上の構成で製品資料に反映する（Product Book自体は本書執筆時点で未作成。作成時にこの構成を用いる） |
| Company OS | 下記「Pricing Manager」を通じて、社内向けに①②③を選択・管理する |
| 営業資料 | ①②③の構成に沿って提案書・提案スライドを作成する |
| 見積書 | 選択された①②③の組み合わせから見積り内容を構成する |
| 契約書 | 見積書の内容と整合する形で契約条件を記載する |

## Pricing Manager（Company OS 新機能・設計）

CEOより、Company OSへ「Pricing Manager」を追加し、営業担当が①利用規模→②プラン→③オプションを選択すると、料金表・見積書・契約書・営業資料を自動生成できる設計とするよう指示があった。本章ではその**設計（データモデル・画面遷移・生成対象）**を定義する。**実際のコード実装は本書の対象外であり、smartlabo-worksリポジトリ側で別途スコープを定めたTaskとしてCEO承認を得たうえで着手する。**契約書の自動生成は特に法的な正確性が求められるため、テンプレート文言の確定にはCEOの確認が必須となる。

### データモデル（設計案）

```
UsageScale   { id: 'small' | 'medium' | 'enterprise', label, description }
Plan         { id: 'lite' | 'standard' | 'premium', label, description, includedFeatures[] }
Option       { id, label, category, status: 'ready' | 'planned' | 'consult' }

ProposalSelection {
  usageScale: UsageScaleId
  plan: PlanId
  options: OptionId[]
  companyName, contactName, ...(見積書・契約書に必要な顧客情報)
}
```

### 画面遷移（設計案）

```
① 利用規模を選択（Small / Medium / Enterprise）
    ↓
② プランを選択（Lite / Standard / Premium）
    ↓
③ オプションを選択（複数選択可）
    ↓
④ 顧客情報を入力（会社名・担当者名 等）
    ↓
⑤ 生成対象を選ぶ
    ├─ 料金表（PDF/画面表示）
    ├─ 見積書
    ├─ 契約書（要CEO確認済みテンプレート）
    └─ 営業資料
```

### 生成対象4種の位置づけ

| 生成物 | 用途 | 備考 |
|---|---|---|
| 料金表 | 顧客への提示用一覧 | ①②③の選択内容を表形式で出力 |
| 見積書 | 正式な見積り | 顧客情報・選択内容・金額を含む |
| 契約書 | 正式契約 | 法的テンプレートの事前確定が必須。CEO確認なしに自動生成しない |
| 営業資料 | 提案・商談資料 | ①②③の構成に沿ったスライド／PDF |

### 実装状況

**未実装（設計のみ）。** `smartlabo-works/app.html`には現状、`PLANS`という異なる形式の静的データ（Starter/Pro/Enterprise、月額固定金額）が存在するが、これは本書の「①利用規模×②プラン×③オプション」構造とは異なり、正式な導入プラン仕様が定まる前の暫定データである。Pricing Manager実装時にこのデータを本書の構造へ置き換える、または移行方針を別途検討する必要がある。

---

## 金額について

本書は**構造（枠組み）を定義するものであり、具体的な金額を定めるものではない**。①②③の組み合わせごとの金額は、`BUSINESS_STRATEGY.md`の「未着手」項目として、今後CEOとの議論により確定する。金額が未確定の間は、ホームページ・営業資料等すべてにおいて「個別相談」「お問い合わせ」の表記を用いる。
