## 1. 私たちは何者か(ポジショニング)

**Smart LaboはAIを売る会社ではありません。会社を進化させる会社です。**

Smart Labo Worksは、中小企業の社長の右腕になるAI経営プラットフォームです。「AIがすごい」ではなく、「この会社なら任せられる」「導入すれば会社が前進する」と感じてもらうことがゴールです。

**ブランドキーワード:** 信頼・未来・成長・経営・AI・DX・プロフェッショナル・革新・安心感

---

## 2. デザインコンセプト: Enterprise AI Platform

目指すのは、世界トップクラスのBtoB SaaSブランドと肩を並べる完成度です。

**デザインキーワード:** Premium / Enterprise / Professional / Minimal / Modern / Trust / Innovation / AI First / Business / Executive / Luxury

最終目標は「かっこいい」ではなく、経営者に**「この会社なら任せたい」「まずはデモを見たい」**と思わせることです。

---

## 3. 参考ブランドと、そこから読み解く思想

Microsoft / OpenAI / AWS / Palantir / Stripe / Datadog / Snowflake / Vercel — これらをそのまま模倣するのではなく(**コピー禁止**)、**共通する思想**を抽出し、Smart Labo独自のブランドへ昇華させます。

| ブランド | 表層的な特徴ではなく、学ぶべき思想 |
|---|---|
| **Microsoft** | エンタープライズが安心して任せられる、揺るぎない信頼感 |
| **OpenAI** | 先進性を、静かな自信として表現するトーン |
| **AWS** | スケールと堅牢さを感じさせるインフラ的な安心感 |
| **Palantir** | ミッションクリティカルな意思決定を支える、重厚で洗練されたデータ表現 |
| **Stripe** | ドキュメントとタイポグラフィの美しさ、開発者・企業からの信頼 |
| **Datadog** | リアルタイムなデータを、上品に、かつ迫力を持って見せるダッシュボード表現 |
| **Snowflake** | クラウド規模を感じさせる、クリーンなブルー基調のエンタープライズサイト |
| **Vercel** | 削ぎ落とされたタイポグラフィと、上品なグラデーションによるモダンさ |

**共通する一本の糸:** これら8社はいずれも、「機能の羅列」ではなく「スケール・信頼・先進性を纏った佇まい」でブランドを語っています。Smart Laboはこの思想を、日本の中小企業経営という文脈に翻訳し、独自の世界観へ昇華します。

## 4. デザインルール

1. **余白を十分に使う** — 情報を整理し、詰め込みすぎない
2. **Premium / Enterprise** — 企業水準の信頼感を、質感・比率で表現する
3. **Professional / Executive** — 経営者に響く風格を保つ
4. **Minimal / Modern** — 削ぎ落とされた、現代的な佇まい
5. **Trust** — 一貫性のあるトーンと品質で、時間をかけて積み上げる
6. **Innovation / AI First** — 先進性を静かな自信として見せる
7. **読みやすさ最優先** — デザインの美しさより先に、文字が読みやすいことを確認する
8. **機能ではなく価値を伝える** — 「何ができるか」ではなく「何が変わるか」を語る

## 5. 使用カラー

| カラー | 役割 |
|---|---|
| **Navy** | 信頼・先進性を表す基調色。背景・重要なセクションに使用 |
| **White** | 明るさ・余白を表現するベース |
| **Smart Blue** | 唯一のアクセントカラー。CTAボタン、リンク、強調要素 |
| **Light Gray** | セクション区切り・補助的な背景 |

- アクセントカラーは **Smart Blueのみ**。
- グラデーションは**上品に**使用する(Navy→Smart Blueなど、洗練された階調のみ。派手な多色グラデーションは禁止)。
- 背景は「明るさ」と「未来感」を両立させる(White基調のセクションとNavy基調のセクションを使い分ける)。
- 具体的なカラーコードが確定次第、[BRAND/](../../BRAND/README.md) に登録し、このファイルからも参照してください。

## 6. 禁止事項(Do NOT)

以下は、Smart Laboのブランドが**明確に拒否する**表現です。AIにデザイン・画像生成を依頼する際は、必ずネガティブプロンプトとしてもこれらを明示してください。

- 安っぽいAIロボットのイラスト・キャラクター
- 六角形(ハニカム)背景
- サイバー演出のやりすぎ(過度なサイバーパンク調・デジタルグリッド)
- ネオンカラー
- 過度な発光・グロー・パーティクルエフェクト
- 情報を詰め込みすぎるデザイン
- チープなイラスト

**重要な注記:** 未来都市・都市ネットワーク・AIネットワークといったモチーフ**自体は禁止ではありません**(次項「推奨するビジュアル」参照)。禁止されるのは、それらを安っぽく、あるいは過度に使うことです。「控えめに」「上品に」が判断基準です。

## 7. 推奨するビジュアル language

- 未来都市、高層ビル、朝日
- 自然光が差し込む洗練されたオフィス
- 都市ネットワーク、AIネットワーク(**控えめに**表現する)
- ガラス、金属などの質感
- 都会的な風景
- 経営者(落ち着きと風格のある佇まい)
- **実際のSmart Labo Works管理画面**(ノートPC・大型モニターに表示された状態)

これらは、「スピード感」と「安心感」を両立する構図で使用してください。

## 8. UI思想(Apple級の完成度、Stripe級の美しさ、Microsoft級の信頼感、OpenAI級の先進性)

- **動き:** ホバーアニメーションなど、意味のあるインタラクションを積極的に使う。ただし上品さを保ち、過剰な演出・自動再生は避ける。
- **ボタン:** **大きめに**、美しい角丸で、ホバーアニメーション付き。高級感を演出する。営業導線を意識し、複数のCTA(無料デモを見る/導入相談をする/資料ダウンロード)を適切に配置する。
- **余白:** 十分に使い、情報を整理すること。
- **タイポグラフィ:** フォントサイズ・行間・コントラストで階層をつくり、信頼感と高級感を両立させる。

詳細な原則は [PROJECT_BIBLE/20_UI_UX_Rules.md](../../PROJECT_BIBLE/20_UI_UX_Rules.md) を参照してください。

## 9. 機能を「価値」として見せる変換パターン

機能をそのまま羅列せず、必ず「得られる価値」に変換して伝えてください。

| 機能 | 伝える価値 |
|---|---|
| メール対応 | 返信漏れを防ぐ |
| CRM | 顧客情報を一元管理 |
| 契約書作成 | 契約作成時間を短縮 |
| 経営分析 | データから意思決定を支援 |

新しい機能を紹介する際は、この表に倣い「機能名 → 価値」の変換を必ず行ってください。

## 10. このファイルの使い方(AIへのプロンプトとして)

Homepage・Dashboard・LandingPageなど、個別の成果物を依頼する際は、このファイルの内容(または要約)を最初にAIへ読み込ませたうえで、個別の指示ファイル([Homepage.md](Homepage.md) / [Dashboard.md](Dashboard.md) / [LandingPage.md](LandingPage.md))を渡してください。

```
1. このファイル(SmartLabo_Design_Bible.md)を読み込ませる
2. 個別の成果物用ファイル(Homepage.md 等)を読み込ませる
3. 具体的な依頼内容を伝える
```
