## 前提

画面を開いた瞬間に**「この会社すごい」**と思わせること。ただし最終目標は「かっこいい」ではなく、**「この会社なら任せたい」「まずはデモを見たい」**と経営者に思わせることです。Apple級の完成度・Stripe級の美しさ・Microsoft級の信頼感・OpenAI級の先進性を目指してください。

## 各セクションの指示

### Hero

営業につながる導線を重視した、左右2カラム構成にする。

**左側:**

> **会社を動かすAI。**
> **Smart Labo Works**
> 社長が本来の仕事に集中できる世界をつくる。

**右側:** 実際のSmart Labo Works管理画面を表示したノートPCや大型モニター。

**背景:** 未来都市。都市にはAIネットワークを**控えめに**表現する(禁止事項の「サイバー演出のやりすぎ」に注意)。

**CTA(3つ、営業導線を意識):**
- 無料デモを見る
- 導入相談をする
- 資料ダウンロード

### 社長の1日の業務をAIが支える流れ

社長の典型的な1日(朝のメール確認、顧客対応、契約書確認、経営判断など)を時系列で示し、各場面でSmart Labo Worksがどう支えるかを描く。機能紹介ではなく、体験のストーリーとして見せる。

### 主要機能

このセクションの見出しには**サービスコピー「経営を、ひとつにつなぐ。」**を使用する。補足の説明文には**説明コピー「社長の右腕になるAI経営プラットフォーム。」**を添える(コピー階層の詳細は [SmartLabo_Design_Bible.md](SmartLabo_Design_Bible.md) 1.5章)。

機能をそのまま並べず、必ず「価値」に変換して見せる([SmartLabo_Design_Bible.md](SmartLabo_Design_Bible.md) 9章の変換パターンを使用)。

例: メール対応 → 返信漏れを防ぐ / CRM → 顧客情報を一元管理 / 契約書 → 契約作成時間を短縮 / 経営分析 → データから意思決定を支援

### 導入メリット

時間・信頼・成長という観点でメリットを整理する。数値・実績は誠実さの原則([PROJECT_BIBLE/00_Foundation/05_Value.md](../../PROJECT_BIBLE/00_Foundation/05_Value.md))に基づき、裏付けのあるもののみ掲載する。

### 対応業種

Smart Labo Worksが対応する業種を紹介する(不動産等、具体化している領域から)。今後の対応拡大も示唆する構成にする。

### Mission

[PROJECT_BIBLE/00_Foundation/03_Mission.md](../../PROJECT_BIBLE/00_Foundation/03_Mission.md) / [04_Vision.md](../../PROJECT_BIBLE/00_Foundation/04_Vision.md) を正として、簡潔に掲載する。

### CTA

Heroと同じ3つのアクション(無料デモを見る/導入相談をする/資料ダウンロード)を、ページ最後にも改めて提示する。

### Footer

会社情報、法務ページ([LEGAL/](../../LEGAL/README.md))へのリンク、問い合わせ導線を配置する。

## UI・ビジュアルチェックリスト

- [ ] Heroは左テキスト・右ビジュアル(管理画面)の構成になっているか
- [ ] Heroの背景に未来都市+控えめなAIネットワークが表現されているか
- [ ] 使用カラーはNavy / White / Smart Blue / Light Grayの範囲内か
- [ ] グラデーションは上品か
- [ ] 禁止ビジュアル(安っぽいロボット・六角形・過度なサイバー演出・ネオン)を使っていないか
- [ ] ボタンは大きめ・美しい角丸・ホバーアニメーション付きか
- [ ] CTAが「無料デモを見る/導入相談をする/資料ダウンロード」の3つ用意されているか
- [ ] 各機能が「価値」として語られているか(機能の羅列になっていないか)
- [ ] 各セクションの余白は十分か
- [ ] スマートフォンでも情報が詰め込みすぎに見えないか
